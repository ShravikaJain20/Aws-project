# Aws-project
 Project 1: Manage and Maintain Web Server Load Using Load Balancer with Auto Scaling (AWS)

 Description

This project demonstrates how to manage and maintain web server traffic efficiently using AWS Load Balancer and Auto Scaling services. The main objective is to distribute incoming user requests across multiple web server instances and automatically adjust the number of servers based on traffic demand.

The Load Balancer helps in evenly distributing network traffic among EC2 instances, preventing any single server from becoming overloaded. Auto Scaling automatically increases the number of running instances during high traffic periods and decreases them during low traffic periods, ensuring optimal performance and cost efficiency.

This architecture provides high availability, fault tolerance, scalability, and better resource utilization. It helps maintain application performance while reducing downtime and manual intervention.

Features

* Traffic distribution using AWS Load Balancer
* Automatic scaling of EC2 instances
* High availability and fault tolerance
* Improved performance under varying traffic loads
* Cost optimization through dynamic resource allocation
* Reduced manual server management

Technologies Used

* AWS EC2
* Elastic Load Balancer (ELB)
* Auto Scaling Group (ASG)
* Amazon CloudWatch
* Virtual Private Cloud (VPC)

## Outcome

Successfully created a scalable and reliable cloud infrastructure capable of handling fluctuating web traffic while maintaining system stability and performance.
